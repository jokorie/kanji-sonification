Project one-pager: Japanese Calligraphy → Sound (stroke-by-stroke)
Vision (v1)

While writing a kanji, the system generates audio that reflects stroke order and motion so the user can learn and recall the character “by its sound.” Each stroke becomes a recognizable sonic gesture; the complete kanji “plays” like a short motif.

Why this is feasible

Hoff (DAFx-18) already showed that kanji gestures can drive sound using Leap Motion/Wacom and two mapping modes—metaphorical (meaning-based) and objective (gesture-based). We’ll focus on objective mappings for real-time feedback and recognizability. 
Jullian Hoff

Apple Pencil exposes rich per-point data (x, y, force, azimuth, altitude, timestamp) via PencilKit (PKStrokePoint/PKDrawing). We can record these and sonify offline now, and stream live later.

Key concepts (short)

Objective sonification (our v1): sound reflects motion features (position, speed, pressure, direction, curvature). This is what Hoff used his additive engine for—neutral, comparable, and responsive. 
Jullian Hoff

Granular vs. Additive (terminology):

Granular = manipulate tiny grains of recorded audio (great for metaphorical “fire/water/bird” textures).

Additive = sum controllable sine partials (great for clean, analytical motion-to-sound).
We’ll prioritize procedural/additive-style control first, then experiment later. (Background from Hoff; general synthesis practice.) 
Jullian Hoff

Recommended v1 mappings (intuitive + memorable)

Primary voice (melodic, for recognizability)

Pitch ← Y (higher on screen → higher pitch). (Keep this stable across experiments.)

Amplitude ← pressure (heavier pressure → louder).

Vibrato depth/rate ← instantaneous speed (faster → more modulation).

Stereo pan ← X (left/right spatial cue).

Stroke-level structure

Stroke boundaries (pen up/down) → note boundaries.

Stroke index (1,2,3,...) → tiny timbral shifts (e.g., partial mix or subtle per-stroke envelope shape) so order is audible.

Kanji-level structure

Fixed tempo grid that advances with horizontal progression; brief rests for inter-stroke pauses.

These choices align with gestural transparency highlighted by Hoff (audience can “hear” the motion they see). 
Jullian Hoff

What’s explicitly out of scope (for v1)

Metaphorical/semantic sample synthesis (granular birds/fire/water). That’s v2+ (requires curated sample banks and different mapping logic). (Hoff’s metaphorical mode.) 
Jullian Hoff

Full iPad live app (we’ll prototype logging later; for now consume exported PencilKit JSON). 
Apple Developer

Machine-learned mappings (DDSP/NNs); first we validate human-interpretable mappings.

Automatic kanji recognition or correctness scoring; v1 is for sonic memory, not grading.

Risk notes & mitigations

Noisy X jitters → pitch wobble: lock pitch to Y primarily; low-pass filter Y before mapping; keep any X→pitch term small (≤0.1).

Clipping/loudness: global limiter in pyo; conservative per-partial gains. 
belangeo.github.io

Timing consistency (offline render): use recorded timestamps; resample to fixed audio rate; stroke onsets quantized lightly so order cues pop.

Developer checklists
Feature extraction (kinematics)

 Normalize (x,y) to [0,1] using canvas bounds.

 Compute speed v = hypot(dx,dy)/dt; smooth with EMA.

 Direction θ = atan2(dy,dx); quantize to 8 bins if needed for “directional motifs.”

 Curvature via change of direction per unit length.

 Segment strokes: pen-down/pen-up from PencilKit strokes.

Sonifier (additive baseline in pyo)

 Boot server before building graph; stop on exit. 
DEV Community

 One oscillator voice per active stroke; or single voice updated by current stroke.

 Map: pitch = lerp(220, 880, 1 - yN), amp = clamp(force*k), pan = (xN*2-1), vib = speed→(depth,rate).

 Short attack on stroke start; audible “tick” optional to encode order.

 Hard limiter + soft clip to avoid spikes.

MIDI engine (mido)

 On stroke start: note_on with pitch from first point; CCs for pressure/speed over time; note_off at stroke end. 
mido.readthedocs.io

Configurable mapping presets

 YAML/JSON with scalar weights & ranges (e.g., pitch_y_weight, pitch_x_weight, vib_depth_max).

 Hot-reload to A/B while auditioning.

Quick pseudocode (offline render)
strokes = load_pencilkit_json(path)      # io/load_pencilkit.py
feat = extract_features(strokes)         # features/kinematics.py

son = AdditiveSonifier(config="presets/a.json")  # engines/additive_pyo.py
son.begin_kanji({"id": "水"})
for s in feat.strokes:
    son.start_stroke(s.summary)
    for pt in s.points:
        son.update(pt, pt.features)
    son.end_stroke()
son.end_kanji()

Success criteria for v1

Given a small set of beginner kanji (e.g., 川, 山, 水, 火, 木), listeners can identify at least 60–70% correctly by audio alone after a few exposures (ear-training effect).

Stroke order is audibly unambiguous (onsets and per-stroke timbre tags are clear).

The same kanji drawn differently still sounds recognizably similar (mapping is robust).
